1. 架构总览 核心

Next.js 的后端能力建立在 App Router 之上，核心思路：每个文件夹即一个路由，特殊文件定义行为。

route.ts / route.js

定义 REST API 端点，替代旧的 Pages Router 中的 /api/ 目录

page.tsx (服务端组件)

默认在服务端渲染，可直接访问数据库、文件系统

'use server' Actions

表单提交与数据变更的利器，无需手写 API

middleware.ts

请求拦截器，处理认证、重定向、日志

关键原则：App Router 中，默认所有组件都是 Server Component。只有标注 'use client' 才会在浏览器执行。这意味着你可以在组件里直接写后端逻辑。 
// 典型项目结构
app/
├── layout.tsx          # 根布局
├── page.tsx            # 首页 (服务端组件)
├── api/
│   └── users/
│       ├── route.ts    # GET /api/users, POST /api/users
│       └── [id]/
│           └── route.ts  # GET /api/users/:id
├── auth/
│   ├── page.tsx        # /auth 页面
│   └── login/
│       └── page.tsx    # /auth/login
└── middleware.ts        # 边缘中间件结构

2. Route Handlers 核心

在 route.ts 中导出 HTTP 方法函数来创建 API 端点。支持 GET、POST、PUT、PATCH、DELETE、HEAD、OPTIONS。

// app/api/users/route.ts
import { NextResponse } from 'next/server'

// GET /api/users
export async function GET() {
  const users = await db.user.findMany()

  return NextResponse.json(users)
}

// POST /api/users
export async function POST(request: Request) {
  const body = await request.json()
  const user = await db.user.create({ data: body })

  return NextResponse.json(user, { status: 201 })
}typescript

Request 对象常用方法

方法用途
request.json()解析 JSON Body
request.text()解析纯文本 Body
request.formData()解析表单数据
request.nextUrl.searchParams获取 URL 查询参数
request.headers读取请求头
NextResponse.json() 等价于 new Response(JSON.stringify(data), { headers: {'Content-Type': 'application/json'} })，更简洁。

3. Server Actions 核心

Server Actions 是 Next.js 后端最优雅的特性之一：直接从前端调用服务端函数，无需手写 API 路由。

用户点击 → 触发 Server Action → 服务端执行 → 返回结果/重定向 
// app/actions.ts
'use server'

import { revalidatePath } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string
  const content = formData.get('content') as string

  await db.post.create({ data: { title, content } })

  // 刷新页面缓存 + 重定向
  revalidatePath('/posts')
  redirect('/posts')
}typescript
// app/posts/new/page.tsx — 绑定到表单
import { createPost } from '@/app/actions'

export default function NewPostPage() {
  return (
    <form action={createPost}>
      <input name="title" required />
      <textarea name="content" required />
      <button type="submit">发布</button>
    </form>
  )
}tsx
Server Action vs Route Handler 怎么选？如果操作和 UI 表单紧密关联 → 用 Server Action。如果需要给第三方调用、或返回非 JSON 数据 → 用 Route Handler。

useActionState（推荐模式）

Next.js 15 引入了 useActionState，统一管理状态和错误：

'use client'
import { useActionState } from 'react'
import { createUser } from '@/app/actions'

export default function Form() {
  const [state, formAction, isPending] = useActionState(createUser, null)

  return (
    <form action={formAction}>
      {state?.error && <p style={{color:'red'}}>{state.error}</p>}
      <input name="name" />
      <button disabled={isPending}>{isPending ? '提交中...' : '提交'}</button>
    </form>
  )
}tsx

4. Server Components 核心

App Router 的默认组件类型。在服务端渲染，可以直接调用数据库、读取文件、使用服务端密钥。

// app/page.tsx — 无需 'use client'，默认服务端组件
import { db } from '@/lib/db'

export default async function HomePage() {
  // 直接查库，无需 API
  const posts = await db.post.findMany({
    orderBy: { createdAt: 'desc' },
    take: 10,
  })

  return (
    <main>
      {posts.map(post => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.content}</p>
        </article>
      ))}
    </main>
  )
}tsx

Server Component

可访问数据库、文件系统、环境变量；不能使用 useState/useEffect；不能添加事件监听器

Client Component

需标记 'use client'；可使用 hooks 和浏览器 API；不能直接访问数据库

5. 数据获取模式 核心

场景方案缓存行为
页面初始化加载Server Component 中 async/await默认缓存（可配置）
用户交互后加载Client Component + fetch / SWR / TanStack Query客户端管理
表单提交/数据变更Server ActionsrevalidatePath/Tag
第三方 API 端点Route Handlers默认不缓存
// Server Component 中直接 fetch
export default async function WeatherPage() {
  const res = await fetch('https://api.weather.com/...', {
    next: { revalidate: 3600 }  // ISR: 每小时重新验证
  })
  const data = await res.json()

  return <div>温度: {data.temp}°C</div>
}tsx
fetch 缓存选项：{ cache: 'no-store' } 每次请求重新获取；{ next: { revalidate: N } } N 秒后过期重新验证；默认在 build 时缓存。

6. Middleware 进阶

在请求到达页面或 API 之前执行，适合做认证检查、重定向、国际化路由。

// middleware.ts (项目根目录)
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const token = request.cookies.get('token')?.value

  // 未登录用户访问 /dashboard 时重定向到 /login
  if (!token && request.nextUrl.pathname.startsWith('/dashboard')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
}

// 配置匹配路径（只有匹配的路径才会执行 middleware）
export const config = {
  matcher: ['/dashboard/:path*', '/api/:path*']
}typescript
注意：Middleware 运行在 Edge Runtime，不支持 Node.js 全部 API（如 fs、部分 npm 包）。如需完整 Node 能力，请在 Route Handler 中处理。

7. 动态路由与参数 核心

使用 [参数名] 创建动态路由段。

// app/api/users/[id]/route.ts
export async function GET(
  request: Request,
  { params }: { params: Promise<{ id: string }> }  // Next.js 15 中 params 是 Promise
) {
  const { id } = await params

  const user = await db.user.findUnique({ where: { id: Number(id) } })

  if (!user) {
    return NextResponse.json({ error: 'Not found' }, { status: 404 })
  }

  return NextResponse.json(user)
}typescript
// Catch-all 路由: app/shop/[...slug]/page.tsx
// /shop/a → { slug: ['a'] }
// /shop/a/b/c → { slug: ['a', 'b', 'c'] }注释
Next.js 15 变更：params 现在是 Promise 类型，必须使用 await 解包。这是与 14 版本的主要区别。

8. Cookies 与认证 进阶

// 在 Route Handler 中设置 Cookie
import { cookies } from 'next/headers'

export async function POST(request: Request) {
  const cookieStore = await cookies()
  cookieStore.set('token', 'jwt-value', {
    httpOnly: true,
    secure: true,
    sameSite: 'lax',
    maxAge: 60 * 60 * 24 * 7,  // 7天
    path: '/',
  })

  return NextResponse.json({ message: '登录成功' })
}typescript
// 在 Server Component 中读取 Cookie
import { cookies } from 'next/headers'

export default async function ProfilePage() {
  const cookieStore = await cookies()
  const token = cookieStore.get('token')?.value
  // 用 token 查询用户...
}typescript
认证方案推荐：个人项目用 NextAuth.js（Auth.js v5）；企业项目考虑自研 JWT + Cookie。两者都能和 Server Components / Middleware 无缝配合。

9. 数据库集成 进阶

Next.js 对数据库无绑定，但 Prisma 是社区最主流的选择。

// lib/db.ts — Prisma 客户端单例
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient }
export const db = globalForPrisma.prisma || new PrismaClient()

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = db
}typescript
ORM / 工具特点
Prisma类型安全，声明式 schema，迁移工具完善
Drizzle更轻量，SQL-like API，性能优
Kysely纯 TypeScript SQL 查询构建器
Neon / PlanetScaleServerless 数据库，按请求计费
Serverless 注意：开发中要避免热重载创建多个数据库连接，上面代码中的 globalThis 技巧正是为此。

10. 错误处理 实用

// app/api/users/route.ts
export async function POST(request: Request) {
  try {
    const body = await request.json()

    // 输入验证
    if (!body.name || body.name.length < 2) {
      return NextResponse.json(
        { error: '名称至少 2 个字符' },
        { status: 400 }
      )
    }

    const user = await db.user.create({ data: body })
    return NextResponse.json(user, { status: 201 })

  } catch (error) {
    if (error instanceof SyntaxError) {
      return NextResponse.json({ error: '请求体格式错误' }, { status: 400 })
    }
    console.error('创建用户失败:', error)
    return NextResponse.json({ error: '服务器内部错误' }, { status: 500 })
  }
}typescript

页面级错误边界

// app/error.tsx — 捕获该路由段内所有未处理异常
'use client'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  return (
    <div>
      <h2>出了点问题</h2>
      <p>{error.message}</p>
      <button onClick={() => reset()}>重试</button>
    </div>
  )
}tsx

文件环境是否提交 Git
.env.local本地开发否（gitignore）
.env.development开发环境是（团队共享）
.env.production生产环境是（团队共享）
Server 端可以直接访问所有环境变量；Client 端只能访问以 NEXT_PUBLIC_ 开头的变量。例如：NEXT_PUBLIC_API_URL=https://api.example.com。
