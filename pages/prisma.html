Prisma 是什么？

下一代 Node.js & TypeScript ORM

Prisma 是一个现代化的对象关系映射（ORM）工具，帮助开发者在 TypeScript/Node.js 中以类型安全的方式读写数据库。核心理念是：先定义 Schema，自动生成类型安全的 Client。

核心优势：自动补全、类型推导、声明式数据模型、直观的迁移管理。相比传统 ORM（如 TypeORM / Sequelize），减少约 40% 的样板代码。

前置准备

确保你的环境已就绪

  • 已安装 Node.js 18+（推荐 20+）
  • 已安装 TypeScript（本教程基于 TS）
  • 有 PostgreSQL 数据库（或使用免费的 Prisma Postgres）
ℹ
没有数据库？Prisma 提供免费托管的 Prisma Postgres，可通过 npx create-db 一键创建，零配置启动。
📁

创建项目

初始化一个新的 TypeScript 项目

# 创建项目目录 mkdir hello-prisma cd hello-prisma # 初始化 npm + 安装 TypeScript 工具链 npm init -y npm install typescript tsx @types/node --save-dev npx tsc --init
ℹ
tsx 让你可以直接运行 TypeScript 文件，无需手动编译。
📦

安装 Prisma 依赖

安装 CLI、Client 及数据库驱动

# 安装 Prisma CLI（开发依赖） npm install prisma --save-dev # 安装 Prisma Client + PostgreSQL 驱动 npm install @prisma/client @prisma/adapter-pg pg dotenv # 安装 Prisma CLI（开发依赖） pnpm add prisma -D # 安装 Prisma Client + PostgreSQL 驱动 pnpm add @prisma/client @prisma/adapter-pg pg dotenv # 安装 Prisma CLI（开发依赖） bun add prisma --dev # 安装 Prisma Client + PostgreSQL 驱动 bun add @prisma/client @prisma/adapter-pg pg dotenv
prisma — CLI 工具，提供 prisma init、prisma migrate、prisma generate 等命令
@prisma/client — Prisma Client 运行时库，用于查询数据库
@prisma/adapter-pg — node-postgres 驱动适配器，连接 Prisma Client 与数据库
pg — node-postgres 底层驱动
dotenv — 从 .env 文件加载环境变量
🔌

配置 ESM + 初始化 Prisma

设置模块系统并创建 Prisma 项目结构

首先，更新 tsconfig.json 启用 ESM：

{ "compilerOptions": { "module": "ESNext", "moduleResolution": "bundler", "target": "ES2023", "strict": true, "esModuleInterop": true, "ignoreDeprecations": "6.0" } } { "type": "module" }

然后，初始化 Prisma：

# 初始化 Prisma 项目 npx prisma init --output ../generated/prisma # 创建 Prisma Postgres 数据库（如使用托管数据库） npx create-db
💡
prisma init 会自动创建 prisma/schema.prisma、.env 和 prisma.config.ts。使用 --output 指定 Client 生成路径，避免污染源码目录。
📝

定义数据模型

在 schema.prisma 中声明你的数据结构

打开 prisma/schema.prisma，添加以下模型（经典的 User ↔ Post 一对多关系）：

generator client { provider = "prisma-client" output = "../generated/prisma" } datasource db { provider = "postgresql" } model User { id Int @id @default(autoincrement()) email String @unique name String? posts Post[] } model Post { id Int @id @default(autoincrement()) title String content String? published Boolean @default(false) author User @relation(fields: [authorId], references: [id]) authorId Int }
@id — 标记主键
@default(autoincrement()) — 自增默认值
@unique — 唯一约束
String? — 可选字段（可为 null）
@relation — 定义外键关系，fields 指本表列，references 指目标表列
🔄

运行数据库迁移

将 Schema 变更同步到数据库

# 创建并应用迁移（首次命名为 init） npx prisma migrate dev --name init # 生成 Prisma Client 代码 npx prisma generate
ℹ
migrate dev 做了两件事：
1. 根据 Schema 创建 SQL 迁移文件
2. 在数据库中执行这些迁移

注意：开发阶段用 migrate dev，生产环境用 migrate deploy（不创建新迁移）。
🔗

实例化 Prisma Client

创建与数据库的连接实例

创建 lib/prisma.ts 文件：

import "dotenv/config"; import { PrismaPg } from "@prisma/adapter-pg"; import { PrismaClient } from "../generated/prisma/client"; const connectionString = `${process.env.DATABASE_URL}`; const adapter = new PrismaPg({ connectionString }); const prisma = new PrismaClient({ adapter }); export { prisma };
💡
通过 driver adapter 模式连接数据库是 Prisma 最新的推荐做法。此模式支持连接池优化、边缘运行时（Edge Runtime）等高级场景。
🚀

编写你的第一个查询

增删改查，类型安全，自动补全

创建 script.ts，试试最常用的操作：

import { prisma } from "./lib/prisma"; async function main() { // ✅ CREATE — 创建用户 + 关联文章 const user = await prisma.user.create({ data: { name: "Alice", email: "alice@prisma.io", posts: { create: { title: "Hello World", content: "My first post!", published: true, }, }, }, include: { posts: true }, }); console.log("Created user:", user); // ✅ READ — 查询所有用户及文章 const allUsers = await prisma.user.findMany({ include: { posts: true }, }); console.log("All users:", JSON.stringify(allUsers, null, 2)); } main() .then(async () => { await prisma.$disconnect(); }) .catch(async (e) => { console.error(e); await prisma.$disconnect(); process.exit(1); });

运行脚本：

npx tsx script.ts
// UPDATE — 更新记录 await prisma.user.update({ where: { email: "alice@prisma.io" }, data: { name: "Alice Updated" }, }); // DELETE — 删除记录 await prisma.post.deleteMany({ where: { published: false }, }); // FILTER — 条件筛选 const publishedPosts = await prisma.post.findMany({ where: { published: true }, orderBy: { id: "desc" }, take: 10, });

Prisma Studio

内置的数据库可视化管理工具

Prisma 自带一个类似 Firebase Console 的数据库浏览器，一行命令启动：

npx prisma studio
💡
自动打开 http://localhost:5555，可以可视化地浏览、筛选、编辑、创建和删除数据。非常适合开发调试。
