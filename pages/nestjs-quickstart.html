💎 2. 架构核心概念 核心

NestJS 的灵魂在于它的分层架构和依赖注入。理解这四个核心，你就掌握了 80% 的 NestJS。

Client 请求 → Middleware → Guard → Interceptor → Pipe → Controller → Service → 响应

Controller

处理请求路由、参数解析，调用 Service 获取数据并返回

Service / Provider

业务逻辑层，可被注入到 Controller 或其他 Service 中

Module

组织单元，将 Controller、Service 等打包到一起

Module 是一切的容器

整个应用就是一个根模块 (AppModule)，内部可嵌套子模块

👠 3. Controllers 控制器 核心

Controller 负责处理 HTTP 请求。通过装饰器定义路由、方法和参数绑定。

// cats.controller.ts
import { Controller, Get, Post, Body, Param, Delete, Put } from '@nestjs/common';
import { CatsService } from './cats.service';
import { CreateCatDto } from './dto/create-cat.dto';

@Controller('cats')
export class CatsController {

  @Get()
  findAll() {
    return this.catsService.findAll();
    // 自动序列化为 JSON，状态码 200
  }

  @Get(':id')
  findOne(@Param('id') id: string) {
    return this.catsService.findOne(id);
  }

  @Post()
  create(@Body() createCatDto: CreateCatDto) {
    return this.catsService.create(createCatDto);
  }

  @Put(':id')
  update(@Param('id') id: string, @Body() dto: CreateCatDto) {
    return this.catsService.update(id, dto);
  }

  @Delete(':id')
  remove(@Param('id') id: string) {
    return this.catsService.remove(id);
  }
}typescript
Copy

常用装饰器速览

装饰器绑定位置示例
@Get() / @Post() 等方法定义 HTTP 方法和路径
@Body()参数获取请求体
@Param('id')参数获取路径参数
@Query('page')参数获取查询参数
@Headers('authorization')参数获取请求头
@HttpCode(201)方法自定义响应状态码
@Header('X-Custom', 'val')方法设置响应头
记住：Controller 只做「接收请求 → 调用 Service → 返回结果」。业务逻辑放 Service，保持 Controller 瘦身。

🌸 4. Providers 提供者 核心

Service 是最常见的 Provider。通过 @Injectable() 装饰器标记，可被依赖注入到其他组件中。

// cats.service.ts
import { Injectable } from '@nestjs/common';

@Injectable()
export class CatsService {
  private readonly cats: any[] = [];

  create(cat: any) {
    this.cats.push(cat);
    return cat;
  }

  findAll(): any[] {
    return this.cats;
  }

  findOne(id: string): any {
    return this.cats.find(c => c.id === id);
  }
}typescript
Copy
Provider 不只是 Service，还包括 Repository、Factory、Helper 等任何用 @Injectable() 标记的类。NestJS 用「Provider」作为统一称呼。

🎀 5. Modules 模块 核心

Module 是 NestJS 的组织单元，用 @Module() 装饰器声明自己的 Controller 和 Provider。

🪄 6. 依赖注入 精髓

这是 NestJS 最核心的设计模式。你不需要手动 new 一个 Service，而是通过构造函数让 Nest 自动注入。

// cats.controller.ts — 通过构造函数注入 Service
@Controller('cats')
export class CatsController {

  constructor(private readonly catsService: CatsService) {}

  @Get()
  findAll() {
    return this.catsService.findAll();
  }
}

// TypeScript 的简写等价于：
// private catsService: CatsService;
// constructor(catsService: CatsService) {
//   this.catsService = catsService;
// }typescript
Copy
为什么用依赖注入？方便测试（可以注入 Mock 对象）、解耦（换实现只需改 Provider）、生命周期由容器管理（单例/请求级作用域）。 
// 用 useClass — 替换实现（如测试时用 MockService）
@Module({
  providers: [
    {
      provide: 'CACHE_SERVICE',
      useClass: CacheService,
    }
  ]
})

// 用 useFactory — 需要条件逻辑创建实例
@Module({
  providers: [
    {
      provide: 'CONFIG_SERVICE',
      useFactory: () => {
        return process.env.NODE_ENV === 'prod'
          ? new ProdConfigService()
          : new DevConfigService();
      },
    }
  ]
})

// 用 useValue — 注入常量/配置对象
@Module({
  providers: [
    {
      provide: 'APP_CONFIG',
      useValue: { apiKey: 'xxx', baseUrl: 'https://api.example.com' },
    }
  ]
})typescript

🧚 8. Pipes 管道 进阶

Pipe 在数据到达 Controller 之前进行转换/验证。最常用的是 ValidationPipe。

// main.ts — 开启全局验证
import { ValidationPipe } from '@nestjs/common';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe({
      transform: true,             # 自动将 plain object 转为 DTO 实例
      whitelist: true,             # 自动剥离未定义的属性（防注入）
      forbidNonWhitelisted: true, # 非白名单属性直接报错
    })
  );

  await app.listen(3000);
}typescript
Copy
内置 Pipe用途
ValidationPipe基于 class-validator 验证 DTO
ParseIntPipe将字符串参数转为数字
ParseUUIDPipe验证 UUID 格式
ParseFloatPipe将字符串参数转为浮点数
// 参数级别使用
@Get(':id')
findOne(@Param('id', new ParseIntPipe()) id: number) {
  // id 已经是 number 类型，非法输入会抛出 400 错误

✨ 9. Interceptors 拦截器 进阶

拦截器可以绑定在请求前/后执行逻辑，适合做日志、响应包装、缓存、超时控制等。

// interceptors/logging.interceptor.ts
import {
  Injectable, NestInterceptor, ExecutionContext,
  CallHandler, Logger
} from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  private readonly logger = new Logger('HTTP');

  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const method = request.method;
    const url = request.url;
    const now = Date.now();

    return next.handle().pipe(
      tap(() => this.logger.log(`${method} ${url} ${Date.now() - now}ms`))
    );
  }
}typescript
Copy
// 响应包装拦截器
@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      map(data => ({ success: true, data, timestamp: new Date().toISOString() }))
    );
  }
}

// 使用：app.useGlobalInterceptors(new TransformInterceptor())
// 每个响应自动变成: { "success": true, "data": {...}, "timestamp": "..." }typescript

👑 10. Guards 守卫 进阶

Guard 决定一个请求是否被允许处理。最常见的用途是认证和鉴权。

// guards/auth.guard.ts
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';
import { Reflector } from '@nestjs/core';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const roles = this.reflector.get<string[]>('roles', context.getHandler());
    if (!roles) return true;  # 没设置 roles 则放行

    const request = context.switchToHttp().getRequest();
    const user = request.user;

    return roles.some(role => user.roles?.includes(role));
  }
}

// 使用：@UseGuards(AuthGuard) + @SetMetadata('roles', ['admin'])
// 或自定义装饰器：@Roles('admin') 配合 Reflectortypescript
Copy

🦋 11. Middleware 中间件 进阶

NestJS 中间件和 Express 中间件基本一致，在路由处理前执行。

// middleware/logger.middleware.ts
import { Injectable, NestMiddleware } from '@nestjs/common';
import { Request, Response, NextFunction } from 'express';

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    console.log(`${req.method} ${req.url} — ${new Date().toISOString()}`);
    next();
  }
}

// 在模块中应用
// cats.module.ts
@Module({
  # ...
})
export class CatsModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer
      .apply(LoggerMiddleware)
      .forRoutes('cats');  # 只对 /cats 路由生效
  }
}typescript
Copy
执行顺序：Middleware → Guards → Interceptors(before) → Pipes → Controller → Interceptors(after) → Exception Filters

🧸 14. 异常过滤器 实用

全局异常过滤器统一处理错误响应格式，避免每个 Controller 手写 try-catch。

// common/filters/all-exceptions.filter.ts
import {
  ExceptionFilter, Catch, ArgumentsHost,
  HttpException, HttpStatus
} from '@nestjs/common';
import { Request, Response } from 'express';

@Catch()
export class AllExceptionsFilter implements ExceptionFilter {
  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();

    const status =
      exception instanceof HttpException
        ? exception.getStatus()
        : HttpStatus.INTERNAL_SERVER_ERROR;

    response.status(status).json({
      success: false,
      statusCode: status,
      message: exception instanceof HttpException ? exception.message : 'Internal error',
      path: request.url,
      timestamp: new Date().toISOString(),
    });
  }
}

// main.ts
// app.useGlobalFilters(new AllExceptionsFilter());typescript
Copy
内置异常：BadRequestException、NotFoundException、UnauthorizedException、ForbiddenException。直接 throw new NotFoundException('Cat not found') 即可。
