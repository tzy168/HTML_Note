Next.js 快速入门

从零开始，三分钟搭建你的第一个 React 全栈应用。本教程将带你完成安装、项目创建、路由配置、数据获取与部署全流程。

Terminal 
$ npx create-next-app@latest
0

准备工作

Node.js 18.17 或更高版本

推荐使用 LTS 版本，运行 node -v 检查版本

包管理器

npm、yarn 或 pnpm 均可

React 基础知识

了解组件、Props 和 JSX 语法

1

安装与创建

运行以下命令创建一个全新的 Next.js 项目。安装过程会引导你选择 TypeScript、Tailwind CSS、ESLint 等配置项。

Terminal 
$ npx create-next-app@latest --typescript

点击上方标签切换配置选项，查看完整的命令行参数。安装完成后进入项目目录：

Terminal 
$ cd my-app && npm run dev
2

项目结构

创建完成后，你会看到以下目录结构。App Router 是 Next.js 13+ 的默认路由方案，文件系统即路由。

File Tree 
my-app/
├── app/
│   ├── layout.tsx    // 根布局，包裹所有页面
│   ├── page.tsx      // 首页，对应路由 /
│   ├── globals.css   // 全局样式
│   └── loading.tsx   // 加载状态（可选）
├── public/           // 静态资源
├── next.config.js  // Next.js 配置
├── package.json     // 依赖管理
└── tsconfig.json    // TypeScript 配置

layout.tsx

根布局组件，定义 HTML 结构和共享的 UI 元素，如导航栏和 footer。

page.tsx

页面组件，文件路径即为 URL 路径。app/page.tsx 对应网站的根路径。

loading.tsx

自动加载态组件，Next.js 会在页面加载时自动显示此组件作为 Suspense fallback。

public/

静态资源目录，存放图片、字体等文件，可通过 / 直接访问。

3

路由系统

App Router 基于文件系统，文件夹结构直接映射为 URL 路径。创建一个 page.tsx 即可生成一条路由。

app/page.tsx 
export default function Home() {
  return <h1>首页</h1>
}
app/blog/[slug]/page.tsx 
export default function BlogPost({ params }) {
  return <h1>文章: {params.slug}</h1>
}

[slug] 是动态路由参数，访问 /blog/hello-world 时 params.slug 的值为 hello-world。

嵌套布局模式

在 app/dashboard/layout.tsx 中创建布局，会自动包裹 dashboard 下所有子路由页面：

app/dashboard/layout.tsx 
export default function DashboardLayout({ children }) {
  return (
    <section>
      <nav>侧边栏</nav>
      {children}
    </section>
  )
}
4

数据获取

App Router 的 Server Components 默认在服务端渲染，可以直接使用 async/await 获取数据，无需额外配置。

app/page.tsx 
async function getData() {
  const res = await fetch('https://api.example.com/posts')
  if (!res.ok) throw new Error('请求失败')
  return res.json()
}

export default async function Page() {
  const data = await getData()
  return (
    <ul>
      {data.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}

使用 useEffect 获取数据

在 Client Component 中，需要添加 'use client' 指令：

ClientComponent.tsx 
'use client'
import { useEffect, useState } from 'react'

export default function ClientComponent() {
  const [data, setData] = useState([])
  useEffect(() => {
    fetch('https://api.example.com/posts')
      .then(res => res.json())
      .then(setData)
  }, [])
  return <ul>{data.map(p => <li>{p.title}</li>)}</ul>
}
5

部署上线

Next.js 与 Vercel 深度集成，几条命令即可将项目部署到生产环境。也可选择 Docker、Node.js 服务器或其他平台。

Terminal 
$ npm i -g vercel && vercel

部署完成后，Vercel 会为你提供一个 HTTPS 域名。每次推送到 GitHub 仓库时，Vercel 会自动触发重新部署。其他部署方案包括 next build + next start 自托管、Docker 容器化部署，以及 AWS、Cloudflare 等云平台方案。
