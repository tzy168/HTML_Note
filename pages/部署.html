分步部署指南

按照以下 7 个步骤，将本地 NestJS 项目安全部署到阿里云服务器。每步均可独立完成。

首先通过 SSH 连接到阿里云 ECS 实例，然后安装所有必需的基础软件。

Terminal 
# SSH 连接服务器
ssh root@your-server-ip

# 系统更新
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo systemctl enable docker && sudo systemctl start docker

# 将当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER

# 安装 Nginx
sudo apt install nginx -y

# 配置防火墙
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable

Tip: 阿里云安全组也需要放行 80、443、22 端口，在 ECS 控制台 > 安全组 > 配置规则中操作。

使用多阶段构建，在 builder 阶段编译 TypeScript 并生成 Prisma Client，在 runner 阶段仅保留运行时文件。

Dockerfile 
FROM node:22-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npx prisma generate
RUN npm run build

FROM node:22-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./
COPY --from=builder /app/node_modules/.prisma ./node_modules/.prisma
COPY --from=builder /app/prisma ./prisma
EXPOSE 3000
CMD ["node", "dist/main.js"]
.dockerignore 
node_modules
dist
.git
.env
*.md
test
coverage

Important: Prisma generate 必须在 builder 阶段运行，否则 runner 阶段缺少编译产物。同时确保 COPY Prisma 相关目录。

Prisma 7 带来了重大改进：全新的 prisma-client provider，不再依赖 Rust 编译，bundle 体积缩小 90%，查询速度提升 3 倍。

Prisma 7 关键变化：
- generator provider 改为 prisma-client（非 prisma-client-js）
- 新增 prisma.config.ts 动态配置文件
- Dockerfile build 阶段需运行 npx prisma generate
schema.prisma (generator) 
generator client {
  provider = "prisma-client"
  output   = "../node_modules/.prisma/client"
}

PostgreSQL 18 Docker 服务配置

PG18 Volume 路径变更：PostgreSQL 18 使用 /var/lib/postgresql 而非 /var/lib/postgresql/data。如果使用旧路径会导致数据丢失！

docker-compose.yml (db service) 
db:
  image: postgres:18-alpine
  restart: unless-stopped
  environment:
    POSTGRES_USER: ${DB_USER}
    POSTGRES_PASSWORD: ${DB_PASSWORD}
    POSTGRES_DB: ${DB_NAME}
  volumes:
    - pg_data:/var/lib/postgresql
  ports:
    - "5432:5432"
  healthcheck:
    test: ["CMD-SHELL", "pg_isready -U ${DB_USER}"]
    interval: 5s
    timeout: 3s
    retries: 5

完整的 docker-compose.yml 编排应用和数据库服务。注意 App 端口绑定到 127.0.0.1，仅通过 Nginx 反向代理对外暴露。

docker-compose.yml 
services:
  app:
    build: .
    restart: unless-stopped
    ports:
      - "127.0.0.1:3000:3000"
    environment:
      DATABASE_URL: postgresql://${DB_USER}:${DB_PASSWORD}@db:5432/${DB_NAME}
      NODE_ENV: production
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:18-alpine
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: ${DB_NAME}
    volumes:
      - pg_data:/var/lib/postgresql
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER}"]
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  pg_data:
.env 
DB_USER=myapp
DB_PASSWORD=your-strong-password-here
DB_NAME=myapp_db
Deploy Commands 
# 启动所有服务
docker compose up --build -d

# 查看日志
docker compose logs -f

# 停止服务
docker compose down

Security: 端口 127.0.0.1:3000:3000 绑定仅允许本机访问，外部请求必须经过 Nginx 代理，避免直接暴露应用端口。

创建 Nginx 站点配置，将 80 端口请求代理到本地 3000 端口。

/etc/nginx/sites-available/your-app 
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
Enable & Reload 
# 创建软链接启用站点
sudo ln -s /etc/nginx/sites-available/your-app /etc/nginx/sites-enabled/

# 测试配置语法
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx

WebSocket 支持：Upgrade 和 Connection 头确保 WebSocket 连接可以正常升级。

使用 Certbot 自动获取 Let's Encrypt 免费 SSL 证书，并自动配置 Nginx HTTPS。

Prerequisite: 域名 DNS 已解析到服务器 IP，Nginx 配置已完成，80 端口可正常访问。

Terminal 
# 安装 Certbot
sudo snap install --classic certbot

# 获取 SSL 证书（自动修改 Nginx 配置）
sudo certbot --nginx -d your-domain.com

# 验证自动续期
sudo certbot renew --dry-run

Auto-renewal:Certbot 安装时会自动创建 systemd timer，证书到期前会自动续期，无需手动操作。

部署完成后，通过以下命令验证服务状态，并了解日常运维中常用的排查命令。

Verify Services 
# 查看所有容器状态
docker compose ps

# 访问健康检查接口
curl http://127.0.0.1:3000/health

# 查看 HTTPS
curl -I https://your-domain.com
Daily Operations 
# 查看应用日志
docker compose logs -f app

# 进入数据库
docker compose exec db psql -U myapp -d myapp_db

# 代码更新后重新部署
docker compose up --build -d

# 重启单个服务
docker compose restart app
Common Issues:
- 容器启动失败: docker compose logs app 查看错误
- 数据库连接失败: 检查 .env 和 DATABASE_URL
- Nginx 502: 确认 app 容器正在运行，端口 3000 正常监听
