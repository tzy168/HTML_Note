// Quick Start

快速上手

从安装到 AI 代理调用，五分钟完成 CodeGraph 配置。

1

安装 CodeGraph 必须

运行一键安装脚本，自动下载并配置 CodeGraph。安装器会检测系统中已安装的 AI 代理，自动写入 MCP 服务器配置。

$ curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh
✓ Downloading CodeGraph...
✓ Installing to /usr/local/bin/codegraph
✓ Detected: Claude Code, Cursor
✓ Configure MCP for Claude Code? (Y/n): Y
✓ MCP server configured successfully
2

初始化项目索引 必须

进入项目根目录，运行 init 命令构建知识图谱索引。CodeGraph 会自动识别项目类型并解析源码结构。

$ cd /path/to/project
$ codegraph init -i
✓ Detected Node.js project (package.json)
✓ Parsing 1,247 files... done (0.8s)
✓ Building dependency graph... done
✓ Index saved to .codegraph/codegraph.db
✓ Cursor rules written to .cursor/rules/codegraph.mdc
3

重启 AI 代理 推荐

重启你的 AI 代理（Claude Code、Cursor 等），使 MCP 服务器配置生效。当项目目录中存在 .codegraph/ 文件夹时，AI 代理会自动调用 CodeGraph 工具。

4

开始使用

现在你可以直接向 AI 代理提问代码相关问题，它会自动调用 CodeGraph 的 MCP 工具获取精准上下文。你也可以使用 CLI 手动查询。

$ codegraph query UserService
Found 3 symbols matching "UserService":
→ src/services/UserService.ts (class UserService)
→ src/services/UserService.test.ts
→ src/types/user.ts (interface UserServiceConfig)

// Architecture

工作原理

CodeGraph 通过 tree-sitter 将源代码解析为知识图谱，存储在本地 SQLite 中。

# 数据流
源代码
  ↓ tree-sitter AST 解析
提取节点（函数 / 类 / 方法）和边（调用 / 导入 / 继承 / 实现）
  ↓ 引用解析
函数调用 → 定义，导入 → 源文件，类继承，框架路由
  ↓ 存入本地 SQLite（.codegraph/codegraph.db）
启用 FTS5 全文搜索
  ↓ MCP Server
原生 OS 文件事件监听，2秒防抖增量同步
i

零配置设计：自动识别语言（基于文件扩展名）、自动排除依赖目录（node_modules、vendor、dist 等）、自动尊重 .gitignore、自动排除大于 1MB 的文件。如需调整排除规则，只需修改 .gitignore。
