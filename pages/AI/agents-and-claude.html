01它们到底是什么

当 AI 助手开始工作时，它会自动读取仓库里的这些约定文件，作为最高优先级的“常驻上下文”。你不必在每条消息里重复说明项目用了什么框架、怎么跑测试——写一次，处处生效。

📌

持久化记忆

把项目背景、约定、踩坑经验沉淀到文件，避免每次重新解释。

🎯

统一行为

团队成员与不同 AI 工具读取同一份规范，输出风格保持一致。

⚙️

降低出错

明确技术栈、命令与边界，减少 AI “自由发挥”导致的破坏性改动。

02CLAUDE.md vs AGENTS.md

两者格式几乎相同，差别主要在“面向谁”。简单说：CLAUDE.md 是 Claude 专属，AGENTS.md 是面向所有 AI 工具的开放标准。

CLAUDE.md

Claude 专属约定

/CLAUDE.md · 由 Claude Code 读取
  • 专为 Claude / Claude Code 设计的指令文件
  • 支持子目录级 CLAUDE.md，按模块就近生效
  • 常写入项目概览、命令、架构、约定与注意事项
  • 可通过 /init 自动生成初稿
  • Claude 在每次会话开始时优先加载
AGENTS.md

开放通用标准

/AGENTS.md · 跨工具共识
  • 面向各类 AI 编码代理的开放、中立规范
  • 被多家工具采用，避免被单一厂商绑定
  • 内容结构与 CLAUDE.md 高度兼容
  • 可让 CLAUDE.md 直接引用 AGENTS.md 复用内容
  • 更适合多 AI 工具混用的团队

实务建议：以 AGENTS.md 作为唯一事实来源（single source of truth），再用一行 CLAUDE.md 引用它，即可同时兼顾通用性与 Claude 的专属优化。

AGENTS.md 
# Project Overview
教学评价系统 — PC 端高校课程评价与统计分析平台。

## Tech Stack
- React 16.14（类组件 + hooks，兼容 IE10+）
- Ant Design 3.26（antd 包，非 @ant-design）
- Dva 2.x（Redux + Saga）· Webpack 5 · TypeScript 5.7

## Commands
yarn start      # 启动开发服务器
yarn build      # 生产构建
yarn lint       # 代码检查
yarn test       # 运行测试

## Conventions
- 组件用 PascalCase，工具函数用 camelCase
- 严禁随意引入新依赖（控制包体积）
- 公共组件 Props 变更须向下兼容，不删除已有字段

## Do Not
- 不要提交个人 API 地址到 public/config.js
- 不要大面积格式化无关代码

04编写最佳实践

把文件当作活文档维护——它越准确，AI 的产出越可靠。

① 精炼而非冗长

只写 AI 推断不出来的关键信息。过长的文件会稀释重点，也消耗上下文。

② 命令必须可复制即用

列出真实可运行的脚本（启动、构建、测试、检查），避免 AI 猜测命令名。

③ 明确“红线”与边界

用 Do / Do Not 清单写明禁止事项，如兼容性要求、不可提交的文件、破坏性操作。

④ 随项目演进持续更新

技术栈或约定变化时同步修订；把反复纠正 AI 的点固化进文件。

⑤ 善用分层与引用

根目录放全局约定，子目录放局部规则；用 AGENTS.md 作单一来源，CLAUDE.md 引用之。
