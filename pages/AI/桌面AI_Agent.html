Part 01

五大桌面 Agent 速览

从字节全产研覆盖的工作台，到 Cursor 以 Agent 为核心重构 IDE，下面是 2026 年最具代表性的五款产品。

Trae SOLO 独立端

字节跳动 / ByteDance

三端打通 限时免费

从 IDE 剥离的跨端 Agent 工作台，App / Web / 桌面三端同步，无需邀请码即可调度云端或本机 Agent 办事。

核心能力 (2026.05)
  • 手机语音跨端操控：5 月 7 日移动端全量上线，发任务后由云端或本机 Agent 执行，你操控的是 Agent 而非屏幕。
  • 双模式智能体：Code 模式（编码，桌面/网页同步，休眠后云端任务不中断）与 MTC 模式（文档、数据、方案全产研场景）。
  • 定时任务 + Skills 市场：可设定时自动执行，技能安装即用、零配置自动调用；新增 Windows 客户端。

OpenAI Codex 桌面端

OpenAI

锁屏托管 Computer Use

由 Computer Use 驱动的 macOS 桌面 Agent，可直接读取屏幕并点选、输入、操作本地 GUI 应用。

核心能力 (2026.05)
  • 锁屏后台运行：5 月 21 日上线，手机发任务后即便屏幕熄灭锁定也能临时解锁执行，全程覆屏防窥，检测到本地键鼠输入立即重锁（合盖休眠除外）。
  • 手机遥控 + Codex for Chrome：手机端审批并查看执行画面；新增 Chrome 扩展，可在浏览器内测试 Web 应用、跨标签取上下文。
  • 权限可控：需装 Computer Use 插件并授予录屏/辅助权限，逐 App 授权；暂不支持 EEA/英国/瑞士，且不操作终端与系统管理员弹窗。

Claude Cowork / Code

Anthropic

MCP 原生 Win & Mac

把 Claude Code 的代理能力带到桌面的知识工作 Agent，5 月 24 日在 Windows / macOS 正式 GA。

核心能力 (2026.05)
  • Computer Use 公测：Win 与 Mac 双端，无连接器时直接点选、输入、开应用操作本地 GUI（Pro/Max 限定，研究预览）。
  • 跨端发起任务：可用手机 app 在外发起并查看，但桌面需保持唤醒、app 保持打开——电脑休眠任务即停止。
  • MCP 生态 + Claude Code：MCP 协议发起者，企业连接器无缝集成；CLI 支持 `--worktree` 隔离与多会话并行。

Marvis 马维斯

腾讯 / Tencent

OS 级控制 每日 1000 万 Token

腾讯应用宝团队打造的操作系统层级 AI 助手，5 月 20 日 Win/Mac/安卓三端上线，一句话操控整台电脑。

核心能力 (2026.05)
  • 1 主 + 5 副 Agent 协同：主 Agent 拆解任务，分派 File / Computer / App / Browser / Search 副 Agent，7×24 全流程闭环。
  • 系统级深度操控：查硬件/电池/网络，调系统设置、清理自启、一句话关 Windows 广告；按人像/主题建 AI 图库与文档库。
  • 双模式：效率模式端云协同，隐私模式端侧模型 0 上云、断网可用；每日免费 1000 万 Token（未来支持自带 Key）。

Cursor 3 / Composer 2

Anysphere

Agents Window 极致性价比

4 月 2 日发布的 Cursor 3.0 把 IDE 重构为以 Agent 为核心的 Agents Window，多 Agent 并行成为默认。

核心能力 (2026.05)
  • 多 Agent 并行：本地、worktree、云端沙箱、远程 SSH 各开独立 tab 并行；云端沙箱关机也能持续运行。
  • /best-of-n 多模型赛马：编辑器内 `/best-of-n sonnet,gpt,composer` 同任务多模型各跑独立 worktree，父 Agent 点评后择优。
  • 自研 Composer 2：基于 Kimi K2.5，Terminal-Bench 2.0 得 61.7；价格 $0.50/M 输入、$2.50/M 输出。

三维演进：传统 IDE、AI CLI 终端与桌面 Agent

传统 IDE（以编辑器为中心）
  • 粒度：以行/函数为单位，被动补全与静态建议。
  • 交互：用户主导，代码需人工逐行编写。
  • 运行：本地单点编辑，关机休眠即暂停。
  • 权限：受沙箱约束，难以跨软件或接管系统。
AI CLI 终端（以命令行为中心）
  • 粒度：以指令/命令为单位，做编译、测试、重构。
  • 交互：终端高频即时反馈，下达专项任务。
  • 运行：可用 `--worktree` 隔离分支，多在本地终端。
  • 权限：限于 Shell 与代码库，不操作复杂 GUI。
桌面 AI Agent（以任务委派为中心）
  • 粒度：以任务/需求为单位，自主规划与纠错。
  • 交互：开发者化身管理者，自然语言描述交付。
  • 运行：云端沙箱可关机续跑；本地 Computer Use 多需保持唤醒。
  • 权限：经 Computer Use 接管 GUI、控制锁屏与手机遥控。

2026 年，桌面 AI Agent 已走出“代码助手”阶段，迈入主动操控、锁屏托管与多端协同的时代。你管理的，不再是代码字符，而是数十个并行、随时可接管的 Agent 工作流。

Part 03

2026 桌面 Agent 四大趋势

随着 5 月腾讯、字节与 OpenAI 的密集发布，桌面 Agent 的技术路径已愈发清晰。

01
📱

无屏操控与跨端分离

Trae SOLO 手机语音发任务、Codex 手机发任务、Marvis 跨端接管——2026 年，人们开始把手机当作 Agent 遥控器，不必再紧盯电脑屏幕。

02
🔒

锁屏与云端托管

Codex 锁屏时临时解锁并覆屏、检测本地输入即重锁；Cursor 与 Trae 的云端沙箱可关机续跑，Agent 获得“夜间生产力”。而本地 Computer Use（如 Claude Cowork）仍需保持唤醒——续航与安全成为分水岭。

03
💻

下沉至 OS 与芯片级

腾讯马维斯把 Agent 从插件层推向操作系统与芯片层（与英特尔、微软 WinML 合作），可直接改系统设置、诊断硬件；端侧模型本地运行、0 上云的隐私模式与端云效率模式双轮驱动。

04
🪙

Token 经济学催生并发

Cursor 把 Composer 2 价格打到 $0.50/$2.50 每百万 Token，Marvis 每日送 1000 万免费 Token，多 Agent 的成本障碍被扫清。以微小成本调度多路并行沙箱，让“Agents 并发”成为 2026 核心指标。
